Политика конфиденциальности

Дата вступления в силу: 8 июля 2026

1. Общие положения

1.1. Настоящая Политика определяет порядок обработки информации в связи с использованием веб-инструментов FlowNote, FlowPhoto и FlowVault (далее — Сервисы, Flow Studio).

1.2. Оператор — физическое лицо, владелец репозитория flow-studio.

1.3. Используя Сервисы, Пользователь соглашается с Политикой. При несогласии — необходимо прекратить использование.

1.4. Политика разработана с учётом требований Федерального закона № 152-ФЗ «О персональных данных» в части, применимой к деятельности Оператора. Документ не является юридической консультацией.

2. Ключевой принцип: Оператор не видит содержимое

2.1. Архитектура Сервисов построена по принципу zero-knowledge / шифрования на стороне клиента.

2.2. Оператор не является получателем персональных данных и иной информации, содержащейся внутри зашифрованного Контента Пользователя (тексты заметок при включённом шифровании, расшифрованные изображения, содержимое Ciphertext), поскольку не располагает ключами расшифровки и технически не может ознакомиться с таким содержимым.

2.3. Оператор не осуществляет модерацию содержания внутри Ciphertext и не формирует публичный доступ к Контенту Пользователя.

3. Категории обрабатываемой информации

3.1. В зависимости от Сервиса Оператор может обрабатывать:

  • Технические данные: IP-адрес, User-Agent, дата и время запроса, идентификаторы объектов, размер файла, срок хранения, счётчик просмотров.
  • Данные аутентификации FlowVault: криптографический хеш пароля сейфа (не сам пароль).
  • Ciphertext: зашифрованные байты файлов — без возможности прочтения содержимого Оператором.
  • Обращения Пользователя: содержание сообщений в GitHub Issues (если Пользователь сам их раскрывает).

3.2. Оператор не обрабатывает специальные категории персональных данных целенаправленно и не извлекает их из Ciphertext.

4. FlowNote — обработка данных

4.1. Содержимое заметок и идей по умолчанию хранится только в браузере Пользователя (localStorage). Оператор не получает это содержимое на сервер.

4.2. При включении функции шифрования мастер-пароль, ключи и recovery-фраза обрабатываются на устройстве Пользователя. Оператор не хранит мастер-пароль.

4.3. Экспорт JSON выполняется локально. Оператор не контролирует дальнейшее хранение экспортированных файлов.

4.4. Страница FlowNote загружается с GitHub Pages. Провайдер (GitHub / Microsoft) может обрабатывать IP-адрес и метаданные HTTP-запроса как самостоятельный оператор/обработчик — см. их политики конфиденциальности.

4.5. Правовое основание (152-ФЗ): обработка технических метаданных при загрузке статической страницы — законный интерес / исполнение пользовательского запроса к сайту; содержимое заметок Оператором не обрабатывается.

5. FlowPhoto и FlowVault — обработка данных

5.1. Изображения шифруются в браузере Пользователя (AES-GCM). Ключ передаётся в фрагменте URL после # и не отправляется на сервер Оператора.

5.2. На сервере хранится Ciphertext и метаданные (имя файла, MIME-тип, размер, срок хранения, идентификатор, счётчик просмотров, опциональный хеш пароля ссылки).

5.3. FlowVault: на сервере — хеш пароля сейфа и привязка идентификаторов фото; ключи расшифровки — у Пользователя (браузер, файл .flowvault).

5.4. Для ограничения злоупотреблений обрабатываются IP-адрес и частота запросов (rate limiting).

5.5. Правовое основание (152-ФЗ): исполнение пользовательского соглашения (Условия использования); законный интерес — обеспечение безопасности и работоспособности Сервиса.

6. Цели обработки

  • предоставление функционала Сервисов (хранение Ciphertext, выдача по приватной ссылке);
  • обеспечение безопасности (HTTPS, rate limiting, квоты);
  • резервное копирование технической базы данных (без доступа к расшифрованному содержимому);
  • исполнение обязательных требований закона (в объёме, предусмотренном законом).

Оператор не продаёт данные, не использует их для рекламного таргетинга и не строит поведенческие профили.

7. Трансграничная передача

7.1. FlowPhoto размещён на инфраструктуре Render (США). Резервные копии могут храниться в Storj (распределённая сеть; узлы могут находиться за пределами РФ).

7.2. Загружая Ciphertext, Пользователь соглашается на трансграничную передачу технических данных и Ciphertext в указанных инфраструктурах.

7.3. FlowNote (GitHub Pages) также может обрабатываться инфраструктурой за пределами РФ.

8. Сроки хранения

8.1. FlowPhoto: Ciphertext и метаданные хранятся до истечения срока, выбранного Пользователем, до удаления Пользователем, в режиме «Секрет» — до первого просмотра, либо до удаления по техническим правилам Сервиса. Резервные копии базы данных ротируются (хранится ограниченное число последних копий).

8.2. FlowNote: данные в браузере хранятся до очистки Пользователем или удаления данных браузера.

8.3. Технические логи хостинг-провайдеров — в сроки, установленные провайдерами.

9. Меры безопасности

Шифрование на клиенте, HTTPS, хеширование паролей FlowVault, ограничение запросов, минимизация собираемых данных. Оператор не гарантирует абсолютную защиту. Пользователь обязан обеспечивать сохранность паролей, ключей в ссылках и файлов бэкапа.

10. Права субъекта персональных данных

10.1. В отношении данных, которые Оператор фактически обрабатывает (метаданные, хеши), Пользователь вправе:

  • запросить уточнение, блокирование или уничтожение — в пределах, допускаемых законом и техническими возможностями;
  • отозвать согласие — путём прекращения использования Сервисов;
  • удалить Ciphertext и учётные записи FlowVault доступными средствами Сервиса.

10.2. Оператор не может предоставить доступ к содержимому внутри Ciphertext или расшифрованных заметок, поскольку не владеет ключами.

10.3. Запросы направляются на контакт из раздела 13. Срок ответа — до 30 календарных дней, если иное не предусмотрено законом.

11. Передача третьим лицам

11.1. Оператор может привлекать обработчиков (хостинг Render, Storj, GitHub) исключительно для технического размещения и резервного копирования.

11.2. Передача возможна по обязательному требованию уполномоченных органов в случаях, предусмотренных законодательством РФ.

12. Изменения Политики

Актуальная версия публикуется на данной странице. Существенные изменения могут сопровождаться обновлением даты вступления в силу. Продолжение использования после публикации изменений означает согласие, если иное не запрещено законом.
