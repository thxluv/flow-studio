РЕШЕНИЯ ПОЛЬЗОВАТЕЛЯ (2026-07-07)
---------------------------------
- Self-destruct: заметки + идеи (Этап 2.5)
- Tor-friendly / офлайн-бандл: отложено (п.5), не приоритет сейчас
- Ложный пароль, vaults и прочее «жёсткое» — только после Этапа 4.1

--------------------------------------------------------------------------------
1. PLAUSIBLE DENIABILITY (ложный пароль)
--------------------------------------------------------------------------------
- Два пароля: ложный открывает «чистое» приложение, настоящий — секретные данные
- Как VeraCrypt: невозможно доказать наличие скрытого тома
- Нужна отдельная архитектура vault (два независимых зашифрованных контейнера)
- Риск: очень высокий, миграция данных, UX сложный

--------------------------------------------------------------------------------
2. НЕСКОЛЬКО СВОДОВ (VAULTS)
--------------------------------------------------------------------------------
- «Личный», «Секретный», «Рабочий» — каждый со своим паролем
- Быстрое переключение между сводами
- Риск: высокий — переписать слой Security и UI навигации

--------------------------------------------------------------------------------
3. ЗАШИФРОВАННАЯ ПЕРЕДАЧА
--------------------------------------------------------------------------------
- Генерация ссылки/текста для другого человека
- Расшифровка только с паролем получателя
- Можно через AES + base64 в URL hash или отдельный файл

--------------------------------------------------------------------------------
4. АУДИТ-ЛОГ
--------------------------------------------------------------------------------
- Журнал: открытие/изменение заметок, блокировки, экспорт
- Хранить зашифрованно отдельно от основного vault

--------------------------------------------------------------------------------
5. TOR-FRIENDLY / ОФФЛАЙН-БАНДЛ
--------------------------------------------------------------------------------
- Убрать внешние CDN: Tailwind, Font Awesome, Google Fonts, jsPDF
- Встроить всё локально в один HTML или папку assets/
- Работа через Tor Browser без внешних запросов
- Риск: средний (размер файла, обновления библиотек)

--------------------------------------------------------------------------------
6. DESKTOP ЧЕРЕЗ TAURI
--------------------------------------------------------------------------------
- Отдельный проект, не один HTML
- Системная ключница, биометрия, изоляция процессов

--------------------------------------------------------------------------------
7. PDF С ПАРОЛЕМ
--------------------------------------------------------------------------------
- Защищённый экспорт PDF (сейчас только JSON с паролем)
- jsPDF не поддерживает нативно — нужна другая библиотека

--------------------------------------------------------------------------------
8. ГОРЯЧИЕ КЛАВИШИ (ЭТАП 2 — ДОРАБОТКА)
--------------------------------------------------------------------------------
- HOTKEYS_ENABLED = false (заглушка)
- Браузер перехватывает Ctrl+K, Ctrl+N и др.
- Нужно: capture-фаза, альтернативные сочетания, UI Win/Mac

--------------------------------------------------------------------------------
9. SELF-DESTRUCT ДЛЯ ИДЕЙ — ✅ СДЕЛАНО (Этап 2.5)
--------------------------------------------------------------------------------
- Реализовано в index.html (idea-destruct-select, id у идей)

--------------------------------------------------------------------------------
10. STEGANOGRAPHY / ZERO-KNOWLEDGE SYNC
--------------------------------------------------------------------------------
- Из плана.txt «жёсткий уровень» — не обсуждалось детально
- Очень высокий риск и неясный UX — только по явному запросу
