1. СУТЬ ИДЕИ
--------------------------------------------------------------------------------

Flow Studio — privacy-first экосистема для личных данных:

  • Пользователь владеет своим контентом и ключами.
  • Оператор (физлицо) технически НЕ видит содержимое — zero-knowledge.
  • Нет регистрации, нет публичной ленты, нет модерации ciphertext.
  • Аудитория: в основном РФ.

Идея в одном предложении:
  «Инструменты, где заметки живут в браузере, а фото шифруются на устройстве
   до отправки — сервер хранит только "кашу", а ключ только у пользователя.»

Позиционирование:
  — Не соцсеть, не ОРИ, не публичная площадка.
  — Pet-project, некоммерческий (кроме будущего FlowNote Pro).
  — Код — интеллектуальная собственность; план: уйти с публичного GitHub.

--------------------------------------------------------------------------------
2. ПРОДУКТЫ
--------------------------------------------------------------------------------

FlowNote (index.html)
  Заметки, идеи, утилиты. localStorage в браузере.
  Опциональное шифрование мастер-паролем на устройстве.
  Хостинг: GitHub Pages → https://thxluv.github.io/flow-studio/

FlowPhoto (flowphoto-server/)
  Приватный обмен фото. AES-GCM 256 в браузере.
  EXIF удаляется ДО шифрования. Имя файла на сервере анонимное (photo_xxx.jpg).
  Ключ — в URL после #, на сервер не уходит.
  Хостинг: Render → https://flowphoto.onrender.com/
  Версия сервера: 3.5.0

FlowVault (/vault на FlowPhoto)
  Личный сейф фото. Пароль → хеш на сервере.
  Ключи расшифровки фото → браузер + файл .flowvault.
  Бэкап БД на Storj (25 ГБ), автозаливка ~60 сек после загрузки.
